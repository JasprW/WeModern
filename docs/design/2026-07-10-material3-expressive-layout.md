# WeModern Material 3 Expressive 布局设计

## 目标

首页只解决四件事：

1. 告诉用户 WeModern 当前是否可用。
2. 给出唯一明确的下一步。
3. 区分必要权限、可靠性优化和高级能力。
4. 在设置完成后提供紧凑的测试入口。

视觉稿见 [PNG 预览](wemodern-m3e-layout.png) 和 [可编辑 SVG](wemodern-m3e-layout.svg)。左侧是首次设置，右侧是全部就绪。

## 当前界面的主要问题

- 状态卡和权限卡重复展示同一组信息，用户需要在两块区域之间比对。
- 已完成权限仍以禁用的大按钮出现，占空间且低对比度看起来像不可用。
- `Live Update` 在不适用的系统版本上也可能显示为失败状态，无法区分“未授权”和“不适用”。
- 同步移除是可选的高级能力，但当前信息量和核心权限相同。
- 三个测试操作使用等权重全宽按钮，让次要功能抢占主要视觉层级。
- 大标题、统一大卡片和统一间距让页面接近三屏长，首屏看不到完整设置流程。

## 新的信息架构

```text
系统状态栏

Top app bar
  WeModern                                      设置

状态焦点区
  待设置: 还差 N 步 + 当前下一步 + 单一主按钮
  已完成: 已就绪 + 正在运行的能力摘要

设置
  通知使用权                     必需  / 已开启
  发送通知                       必需  / 已开启
  Live Update                    推荐  / 已开启
  不受限电池使用                 推荐  / 已开启

高级功能
  同步移除                       未设置 / 已就绪  >
    展开后: 日志读取、调试日志、ADB 命令、重启提醒

测试通知
  [消息] [语音] [视频]

版本信息与故障入口
```

## 首屏布局

### 1. Top app bar

- 高度 64 dp，内容区域左右 20 dp。
- 标题使用 `titleLarge`，不再使用 `displaySmall`。
- 右侧只放一个设置图标，进入系统应用设置。
- 副标题从常驻标题区移除，避免重复解释产品。

### 2. 状态焦点区

这是页面唯一的大面积强调容器。

待设置状态：

- 容器使用 `tertiaryContainer` 或动态色中的暖色容器。
- 标题为“还差 2 步”，使用 `headlineMedium`。
- 正文解释结果：“完成后即可接管微信消息与通话通知”。
- 只展示当前下一步的主按钮，例如“开启通知使用权”。
- 主按钮成功返回后，容器内容切换到下一步，不跳页。

全部就绪状态：

- 容器使用 `primaryContainer`。
- 标题为“已就绪”。
- 正文为“微信通知重写正在运行”。
- 下方只保留能力标签，例如“通知重写”和“Live Update”。
- 不重复列出四个状态，也不显示禁用按钮。

形状建议使用一个有方向性的 Expressive 轮廓：上方两个角 32 dp，左下角 32 dp，右下角 12 dp。形状变化只用于焦点区，普通列表保持熟悉和稳定。

### 3. 设置列表

把权限变成一个连续的分组列表，不再每项套按钮或独立卡片。

每行结构：

```text
[48 dp 图标容器]  面向用户的标题              [状态文字/箭头]
                  一行用途说明
```

- 行高 76 至 84 dp。
- 图标列固定 48 dp，文本起始线保持一致。
- 标题 `titleMedium`，说明 `bodyMedium`。
- 尾部状态不使用禁用 `AssistChip`。已完成使用“对勾 + 已开启”，待处理使用“待开启 + chevron”。
- 第一项和最后一项用较大外侧圆角，中间项只保留 4 dp 间隔，形成明显分组。
- 当前下一步可使用轻微的 `secondaryContainer` 背景，其余行使用 `surfaceContainerLow`。

建议文案：

| 系统能力 | 用户标题 | 用途说明 | 级别 |
|---|---|---|---|
| Notification listener | 通知使用权 | 读取并重写微信通知 | 必需 |
| Post notifications | 发送通知 | 展示重写后的消息与来电 | Android 13+ 必需 |
| Promoted notifications | Live Update | 在状态栏和锁屏显示实时通话进度 | Android 16+ 推荐 |
| Battery optimization | 不受限电池使用 | 降低后台服务被系统暂停的概率 | 推荐 |

`Live Update` 放在设置列表中，但不计入顶部剩余步骤。Android 16+ 时显示“已开启/推荐”，点击后进入专属系统设置；旧版本直接隐藏，不显示失败或不支持状态。

### 4. 高级功能

首页默认只显示一行：

```text
[同步图标]  同步移除
            微信撤回通知时立即同步清理       已就绪  >
```

- 未配置时显示“需 ADB”，但不计入顶部剩余步骤。
- 点击后在当前页面内展开，不使用弹窗。
- 展开内容依次显示“日志读取”“调试日志”两个条件，再显示可复制的命令块和重启提醒。
- 已就绪时只显示汇总状态，具体条件默认折叠。

### 5. 测试操作

- 使用紧凑的三项按钮组或等宽三列按钮：消息、语音、视频。
- 每项高度 56 dp，短标签配不同图标。
- 未完成必要设置时整体不可用，并在标题下显示“完成必要设置后可测试”。
- 触发后使用 snackbar 反馈“测试通知已发送”，避免无反馈重复点击。

## 状态和申请流程

```text
进入首页
  |
  +-- 通知使用权未开 --> 顶部主按钮“开启通知使用权”
  |                         返回 App 后自动刷新
  |
  +-- 通知权限未开 ----> 顶部主按钮“允许发送通知”
  |                         拒绝一次: 保留解释并可重试
  |                         不再询问: 改为“前往应用设置”
  |
  +-- 必要权限完成 ----> 顶部显示“已就绪”
                            Live Update 作为 Android 16+ 推荐项
                            电池优化作为推荐项继续可见
                            ADB 同步移除留在高级功能
```

权限状态至少需要区分：

- `RequiredMissing`：影响核心功能，进入顶部剩余步骤。
- `RecommendedMissing`：不阻塞就绪状态，用“推荐”提示。
- `Granted`：对勾和简短状态文字。
- `NotApplicable`：当前 Android 版本不需要或不支持，使用中性文字。
- `NeedsSettings`：系统权限不再弹窗时，直接进入应用设置。
- `AdvancedManual`：需要 ADB，不混入普通运行时权限。

## 对齐和尺寸规则

- 页面水平边距：手机 20 dp，宽屏 32 dp。
- 内容最大宽度：600 dp，宽屏居中。
- Top app bar 到焦点区：12 dp。
- 焦点区内边距：24 dp。
- 大区块间距：28 dp。
- 标题到内容：12 dp。
- 列表项间距：4 dp。
- 图标容器：48 dp；图标 24 dp。
- 最小触摸目标：48 dp。
- 主按钮高度：56 dp；测试按钮高度：56 dp。
- 文字最多两行，状态区不使用可变宽胶囊堆出右侧锯齿。

## 颜色、字体和动效

- 颜色策略采用克制型动态色：大部分页面使用带主色倾向的中性 surface，仅焦点区和当前动作使用强调色。
- Android 12+ 继续使用动态色。回退配色保留低饱和自然绿，避免整页使用高饱和微信绿。
- 使用系统字体。标题通过字号和字重建立层级，不引入展示字体。
- 状态切换使用 150 至 250 ms 的内容淡入、轻微缩放和颜色过渡。
- 不做页面入场编排；开启减少动态效果时直接切换状态。

## Compose 组件映射

| 布局角色 | 建议组件 |
|---|---|
| 页面骨架 | `Scaffold` + `TopAppBar` + `LazyColumn` |
| 状态焦点区 | `Surface` + 自定义 `RoundedCornerShape` |
| 当前下一步 | 大尺寸 `Button` 或 `FilledTonalButton` |
| 设置列表 | 分组 `Surface` + `ListItem`，尾部自定义状态 |
| 高级功能 | `ListItem` + `AnimatedVisibility` 内联展开 |
| 测试入口 | `ButtonGroup`，或三项等宽 `Row` 回退实现 |
| 操作反馈 | `SnackbarHost` |

使用 `LazyColumn` 替代整页 `Column.verticalScroll`，并将布局状态从六个布尔值整理为派生的 `SetupStep` 和 `CapabilityState`，这样顶部主动作、列表状态和可用性都来自同一份状态模型。

## 验收标准

- 393 dp 宽手机上，顶部状态、三项设置和高级功能入口能在约 1.3 屏内看完。
- 页面上同一权限只出现一次。
- 任意时刻最多只有一个主按钮。
- 用户无需理解 `READ_LOGS` 或系统属性也能完成核心设置。
- TalkBack 能读出标题、用途、必要性、当前状态和可执行动作。
- 字体放大至 200% 时不遮挡尾部状态，必要时状态自动换到第二行。
- Android 8 到 16+ 能正确区分必需、推荐、不适用和高级手动状态。
